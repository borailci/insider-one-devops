insider-one-devops §1 WHAT WE BUILT
§1

What we built

A reproducible production-style slice — no application complexity, all the DevOps surface.

A tiny Go HTTP service exposing four endpoints — /ping, /healthz, /version, and /metrics — packaged in a multi-stage distroless non-root container, deployed to single-node minikube via a Helm chart, served publicly through an Elastic IP-bound EC2 instance in eu-north-1. A GitHub Actions pipeline lints, tests, scans, signs, and attests every image before it lands on GHCR, and rolls the new revision onto the live cluster through SSM Run-Command — no SSH key ever leaves the workflow.

insider-one-devops §6 SERVICE INTERNALS — MAIN.GO
§6

Service internals — main.go

stdlib net/http · no framework, no DI, no codegen.

  • Endpoints — /ping (plain pong), /healthz (200/503 on drain), /version (sha + build_time from ldflags), /metrics (Prometheus exposition).
  • Config — env-only: PORT, LOG_LEVEL, BUILD_SHA, BUILD_TIME.
  • Logs — slog JSON to stdout: ts, level, msg, request_id, plus access log per request.
  • Request-ID middleware — echo X-Request-ID, else UUIDv4.
  • Metrics middleware — http_requests_total{method,path,status} + http_request_duration_seconds.
  • Shutdown — SIGTERM → /healthz returns 503; 10 s drain via context + sync.WaitGroup; exit 0.

Endpoint contract

pathstatusbody
/ping200pong
/healthz200 / 503{"status":"ok|draining"}
/version200{"sha":..,"build_time":..}
/metrics200text/plain; v=0.0.4

Test coverage maps 1:1 to FR/AC IDs in SPEC.md (see main_test.go).

insider-one-devops §7 CONTAINER — DOCKERFILE
§7

Container — Dockerfile

Multi-stage · cross-compile · distroless · nonroot.

  • Builder pinned to $BUILDPLATFORM (native amd64 on GH runner).
  • go.mod / go.sum copied separately → cached layer.
  • go build cross-compiles via GOOS=$TARGETOS GOARCH=$TARGETARCH.
  • -trimpath -s -w -X main.buildSHA / main.buildTime.
  • Final stage = gcr.io/distroless/static-debian12:nonroot (UID 65532).
  • COPY single static binary, no shell, no apt, no glibc.
  • ENTRYPOINT ["/app"] — no flags, env-only config.
FROM --platform=$BUILDPLATFORM \
  golang:1.25-alpine AS builder
WORKDIR /src
COPY go.mod go.sum ./
RUN go mod download
COPY . .
ARG TARGETOS=linux TARGETARCH=amd64
ENV CGO_ENABLED=0
RUN GOOS=$TARGETOS GOARCH=$TARGETARCH \
    go build -trimpath \
      -ldflags "-s -w \
        -X main.buildSHA=$BUILD_SHA \
        -X main.buildTime=$BUILD_TIME" \
      -o /out/app ./
FROM gcr.io/distroless/static-debian12:nonroot
COPY --from=builder /out/app /app
USER nonroot:nonroot
ENTRYPOINT ["/app"]

insider-one-devops §16 WHY THESE CHOICES
§16

Why these choices

Each locked-in decision is committed as an ADR — these are the headlines.

ADR-0001 · TRACK A

Picked Track A (Minikube on EC2 + EIP) because Terraform + AWS OIDC give higher-signal DevOps evidence than a cloudflared / ngrok tunnel demo. Trade-off paid: hourly cost on a non-free-tier account → destroy-when-idle posture.

ADR-0002 · GO + stdlib net/http

Picked Go because distroless static produces a non-root image under 15 MB, first-class Prometheus client, zero runtime dependencies. Stdlib net/http removes a whole class of CVE surface a framework would have added.

ADR-0003 · kubectl set image via SSM

Picked direct rollout over ArgoCD/Flux because the case-study target is a single cluster — GitOps wins on N>1 clusters. SSM keeps port 22 closed and audits every operator command in CloudTrail under the OIDC role.

insider-one-devops §17 CHALLENGES AND FIXES
§17

Challenges and fixes

Every failure mode hit during the build · what fixed it · what it taught us.

Helm template ate apiVersion

{{- if X -}} stripped the newline before apiVersion: when a comment preceded it. kubeconform reported "missing apiVersion" for 5 conditional templates. Fix: drop trailing - on the if-statement.

Multi-arch build cost 5 minutes

QEMU emulation of arm64 Go compilation runs ~10× slower than native. Fix: cross-compile via GOARCH=$TARGETARCH on a native amd64 builder.

Deploy needed a torn-down stack escape hatch

After terraform destroy the AWS_DEPLOY_ROLE_ARN / EC2_INSTANCE_ID secrets were unset and the deploy job failed STS. Fix: preflight step probes both secrets and emits configured=false, downstream OIDC + SSM steps gate on it — workflow stays green when the live stack is intentionally down.

SARIF upload 403 on private repo

upload-sarif requires GitHub Advanced Security on private repos. Fix: continue-on-error: true while private; SARIF resumes when repo flips public.
